# Requirements Document

## Introduction

이 프로젝트는 기존의 aiend AI 분석 엔진을 중심으로 틱데이터를 효율적으로 관리하고, 사용자가 쉽게 데이터를 업로드하고 AI 분석 기능을 활용할 수 있는 통합 플랫폼을 구축하는 것입니다. Backend는 aiend와 frontend 사이의 중개 역할을 하며, 틱데이터 파일 관리, AI 모델 실행, 결과 조회 등의 기능을 제공합니다. Frontend는 사용자 친화적인 웹 인터페이스를 통해 데이터 업로드, 분석 실행, 결과 시각화를 가능하게 합니다.

## Requirements

### Requirement 1

**User Story:** 사용자로서, 틱데이터 파일을 웹 인터페이스를 통해 쉽게 업로드하고 aiend 분석 폴더로 이동시키고 싶습니다. 그래야 복잡한 파일 시스템 조작 없이도 데이터를 분석에 활용할 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 웹 인터페이스에서 틱데이터 파일을 선택하고 업로드 버튼을 클릭 THEN 시스템은 파일을 backend 서버로 전송해야 합니다
2. WHEN backend가 틱데이터 파일을 수신 THEN 시스템은 파일 형식을 검증하고 aiend/data 디렉토리의 적절한 위치로 이동시켜야 합니다
3. WHEN 파일 업로드가 완료 THEN 사용자는 업로드 상태와 파일 위치 정보를 실시간으로 확인할 수 있어야 합니다
4. IF 업로드된 파일이 지원되지 않는 형식 THEN 시스템은 명확한 오류 메시지를 표시해야 합니다

### Requirement 2

**User Story:** 사용자로서, 업로드된 틱데이터에 대해 AI 분석을 실행하고 진행 상황을 모니터링하고 싶습니다. 그래야 분석 작업의 상태를 파악하고 완료를 기다릴 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 분석할 데이터셋을 선택하고 분석 시작 버튼을 클릭 THEN backend는 aiend의 적절한 분석 모듈을 실행해야 합니다
2. WHEN AI 분석이 진행 중 THEN 사용자는 실시간으로 분석 진행률과 상태를 확인할 수 있어야 합니다
3. WHEN 분석이 완료 THEN 시스템은 결과 파일의 위치와 요약 정보를 사용자에게 제공해야 합니다
4. IF 분석 중 오류가 발생 THEN 시스템은 오류 로그와 함께 명확한 오류 메시지를 표시해야 합니다

### Requirement 3

**User Story:** 사용자로서, AI 분석 결과를 웹 인터페이스에서 시각화하고 다운로드하고 싶습니다. 그래야 분석 결과를 쉽게 이해하고 활용할 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 완료된 분석 결과를 선택 THEN 시스템은 주요 지표와 차트를 웹 페이지에 표시해야 합니다
2. WHEN 분석 결과가 표시 THEN 사용자는 예측 정확도, Sharpe Ratio, 수익률 곡선 등의 핵심 지표를 확인할 수 있어야 합니다
3. WHEN 사용자가 결과 다운로드를 요청 THEN 시스템은 분석 결과를 CSV, JSON 등의 형식으로 제공해야 합니다
4. WHEN 여러 분석 결과가 존재 THEN 사용자는 결과들을 비교할 수 있는 대시보드를 이용할 수 있어야 합니다

### Requirement 4

**User Story:** 사용자로서, 다양한 AI 모델과 설정을 선택하여 분석을 실행하고 싶습니다. 그래야 다른 접근법으로 데이터를 분석하고 최적의 결과를 얻을 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 분석을 시작하기 전 THEN 시스템은 사용 가능한 AI 모델 목록(LSTM, Transformer, XGBoost 등)을 제공해야 합니다
2. WHEN 사용자가 특정 모델을 선택 THEN 시스템은 해당 모델의 하이퍼파라미터 설정 옵션을 표시해야 합니다
3. WHEN 사용자가 커스텀 설정으로 분석을 실행 THEN backend는 설정을 aiend에 전달하고 해당 설정으로 모델을 실행해야 합니다
4. IF 설정값이 유효하지 않음 THEN 시스템은 설정 검증 오류를 사용자에게 알려야 합니다

### Requirement 5

**User Story:** 관리자로서, 시스템의 전반적인 상태와 리소스 사용량을 모니터링하고 싶습니다. 그래야 시스템 성능을 관리하고 문제를 사전에 파악할 수 있습니다.

#### Acceptance Criteria

1. WHEN 관리자가 시스템 대시보드에 접근 THEN 시스템은 CPU, 메모리, 디스크 사용량을 실시간으로 표시해야 합니다
2. WHEN AI 분석 작업이 실행 중 THEN 시스템은 현재 실행 중인 작업 목록과 각각의 리소스 사용량을 표시해야 합니다
3. WHEN 시스템 리소스가 임계치를 초과 THEN 시스템은 관리자에게 알림을 전송해야 합니다
4. WHEN 관리자가 작업 관리를 요청 THEN 시스템은 실행 중인 작업을 일시정지하거나 중단할 수 있는 기능을 제공해야 합니다

### Requirement 6

**User Story:** 사용자로서, 과거 분석 이력을 조회하고 관리하고 싶습니다. 그래야 이전 분석 결과를 참조하고 데이터 관리를 효율적으로 할 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 분석 이력 페이지에 접근 THEN 시스템은 날짜, 모델, 데이터셋별로 정렬된 분석 이력을 표시해야 합니다
2. WHEN 사용자가 특정 분석 이력을 선택 THEN 시스템은 해당 분석의 상세 정보와 결과를 표시해야 합니다
3. WHEN 사용자가 분석 결과 삭제를 요청 THEN 시스템은 확인 후 관련 파일과 데이터베이스 레코드를 삭제해야 합니다
4. WHEN 사용자가 분석 결과를 검색 THEN 시스템은 키워드, 날짜 범위, 모델 타입으로 필터링된 결과를 제공해야 합니다
